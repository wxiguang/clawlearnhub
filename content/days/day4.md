## 连接 Gmail 📧
（5分钟快速配置，零基础可落地）

### Step 1：创建 Google Cloud 项目并启用 API
1. 访问 [Google Cloud 控制台](https://console.cloud.google.com)，使用目标 Gmail 账号登录；
2. 点击顶部「选择项目」→「新建项目」，自定义项目名称（如「My AI Assistant」），创建后切换至该项目；
3. 左侧菜单进入「API 和服务 → 库」，搜索并启用以下 API：
   - Gmail API
   - Google Calendar API（提前启用，后续配置日历可直接复用）

> 💡 避坑提示：启用 API 时务必确认顶部项目为刚创建的项目，避免选错导致授权失败。

### Step 2：创建 OAuth 授权凭证（核心步骤）
这一步是为助手生成「合法访问 Google 账号的授权凭证」，按以下步骤操作：
1. 左侧菜单「API 和服务 → 凭证」→ 顶部「创建凭证 → OAuth 客户端 ID」；
2. 首次创建需配置「OAuth 同意屏幕」：
   - 用户类型选「外部」→ 点击「创建」；
   - 仅填写带*项：应用名称、用户支持邮箱、开发者联系邮箱，其余留空，点击「保存并继续」；
   - 「测试用户」环节：点击「添加用户」，输入你的 Gmail 邮箱（必填，否则后续授权报错），保存；
3. 返回「创建 OAuth 客户端 ID」页面，应用类型选「桌面应用」，自定义名称后点击「创建」；
4. 创建成功后，弹出凭证窗口，点击「下载 JSON」，将文件重命名为 `credentials.json`；
5. 将文件放入 OpenClaw 工作目录：`~/clawd/credentials.json`（非技术用户可将文件发送给 AI 助手，由其协助放置）。

### Step 3：一键安装 Gmail 技能
OpenClaw 内置技能市场，一行命令完成安装：
```bash
clawdhub install gog
```
> 💡 关键说明：`gog` 是 Google Workspace 全家桶技能，包含 Gmail + Google Calendar + Google Drive，无需分开安装。

### Step 4：授权与测试
首次调用 Gmail 功能时，系统会自动打开浏览器授权链接，按提示完成 Google 账号登录授权；授权成功后，工作目录会自动生成 `token.json`（免登录凭证，切勿泄露）。

## 安全防护 🔐
接入邮件/日历/浏览器后，助手可访问核心隐私数据，需严格遵守以下安全规则：

### 1. 安全审计命令（必执行）
```bash
# 基础体检：检查常见配置漏洞
openclaw security audit

# 深度审计：覆盖权限/密钥/网络/日志全维度
openclaw security audit --deep

# 自动修复：修复检测到的常规安全问题（审计后执行）
openclaw security audit --fix
```

### 2. 密钥/凭证安全
- 禁止将 API Key、`credentials.json`、`token.json` 提交至 Git 仓库；
- 优先通过环境变量/.env 文件存储密钥，避免明文写入配置；
- 定期轮换密钥（建议每3个月一次），降低泄露风险；
- 给敏感文件设置最小权限：`chmod 600 ~/clawd/token.json`。

### 3. 权限最小化
仅授予助手「必要权限」：
- 仅需读取邮件 → 关闭「发送邮件」权限；
- 仅需查看日历 → 关闭「修改/删除日程」权限。

### 4. 网络安全
- 服务器开启防火墙，仅暴露必要端口；
- SSH 登录禁用密码认证，仅允许密钥登录；
- 定期更新系统：`sudo apt update && sudo apt upgrade`。

### 5. 行为边界定义
在 `SOUL.md` 和 `AGENTS.md` 中明确：
- 需人工确认后才能执行的操作（如发送邮件/修改日程）；
- 禁止外传的敏感数据类型（如身份证/银行卡信息）；
- 直接拒绝执行的风险请求（如访问非法网站/修改系统配置）。

> 💡 安全核心：API Key 不入库、Token 权限最小化、行为边界清晰，可规避99%的隐私泄露风险。
