## 一、什么是 Skills ？
你手机里的应用商店是什么？是各类应用的集合地——需要点外卖装 Uber Eats，需要打车装 Uber，需要看视频装 Netflix。

**OpenClaw 的 Skills 技能系统，就是专属于你的 AI 助手的应用商店。**

每一个 Skill 技能都是一套独立的文件包，适配最新架构的技能通常包含四个核心部分：
- **SKILL.md** —— 技能说明书，向 AI 明确该技能的功能、使用场景和调用方法
- **manifest.json** —— 技能元数据，标注版本、兼容范围、依赖项与权限要求
- **配置文件** —— 存放 API Key、连接参数等授权配置信息
- **脚本文件** —— 技能的具体执行逻辑（简单技能可省略）

安装一个 Skill 技能，本质就是将这套文件放入 `~/clawd/skills/` 目录中。助手启动时会自动扫描并加载所有已安装的技能，新增的懒加载机制会让对话时才唤醒对应技能，大幅降低资源占用，就像手机开机后预加载常用应用、按需唤醒冷门应用一样。

> 💡 **核心逻辑**：AI 的语言理解与逻辑能力已经足够成熟，它真正缺少的是对接外部世界的「工具」，而 Skills 技能系统，就是为它补齐这些工具的核心通道。

## 三、安装你的第一个技能
我们以 **remind-me 定时提醒** 技能为例，这是对新手最友好的入门技能，安装完成后即可立刻上手使用，无复杂配置，还支持多端同步提醒、重复周期设置、模板自定义等进阶功能。

### 方式一：从 ClawHub 一键安装（新手首选）
```bash
clawdhub install remind-me
```
执行命令后，系统会自动从 ClawHub 下载技能包，完成版本兼容校验、依赖安装，并配置到默认的技能目录中：
- 默认工作区目录：`<workspace>/skills/`（通常为当前目录下的 `./skills` 文件夹）
- 全局共享目录：`~/.openclaw/skills`（同一台设备上的多个助手/工作区可复用该技能，添加 `--global` 参数即可安装至此）

### 方式二：手动安装
```bash
# 进入全局技能目录
cd ~/.openclaw/skills
# 克隆技能仓库到本地
git clone https://github.com/openclaw/skill-remind-me remind-me
```
手动安装适合需要自定义修改技能源码、或需要离线安装的场景；如果只是快速上手使用，优先选择 ClawHub 一键安装方式。

### 方式三：从 GitHub 清单挑选安装（ClawHub 替代方案）
如果 ClawHub 网站访问不便，更推荐直接从 GitHub 社区维护的官方技能清单中挑选：
https://github.com/VoltAgent/awesome-openclaw-skills

**使用方法**：
1. 在仓库中按分类找到你需要的技能
2. 将技能名称或仓库链接发送给 AI 助手，让它协助你完成安装与权限验证
3. 安装完成后，让 AI 助手提供 3 条可直接复制使用的指令示例，你只需照着发送即可上手

**重要提示**：绝大多数技能安装完成后无需重启服务，热加载机制会让它在你下一次对话时自动生效。

## 四、新手必装的 10 个高价值技能
以下技能清单按「新手上手收益优先级」排序，先安装 3 个核心必备技能，再根据你的使用场景（开发/建站/运营/内容创作/跨境业务）按需加装。

### 🥇 核心必备级
**1. remind-me — 定时提醒**
日常使用频率极高，一旦用上就无法替代。可以将对话中提到的事项直接转为准时提醒，覆盖会议、账单支付、工作复盘、喝水休息、作息管理等全场景，支持多端推送、重复周期设置、提醒模板自定义。

**2. todo-tracker — 待办清单管理**
将你随口提到的待办事项自动收纳进清单，支持随时查看、标记完成、优先级排序、标签分类，尤其适合多任务并行、需要系统化管理事项的场景，可与日历、提醒技能无缝联动。

**3. Email Kit — 全邮箱智能管理**
替代单一的Gmail技能，兼容Gmail、Outlook、企业邮箱等所有支持IMAP/SMTP协议的邮箱，能让助手实时监控重要邮件、提炼核心要点、草拟回复内容、自动归档分类，帮你从海量邮件中精准筛选核心信息。

**4. Web Search — 联网搜索**
任何需要实时信息的场景都离不开它。没有搜索能力的 AI 助手，就像断网的手机，只能使用离线数据，无法获取最新的行业动态、实时资讯、技术更新等内容，支持多搜索引擎切换、结果精准筛选、来源可信度标注。

### 🥈 强烈推荐级
**5. Browser — 网页操作与信息提取**
让助手打开指定网页、抓取核心数据、对比竞品信息、验证网站可用性（配合 Browser Relay 可实现更强大的功能，需注意网络安全配置），支持动态页面解析、批量内容抓取、页面截图与可用性检测。

**6. weather-pro — 天气与出行提醒**
一句话查询实时天气、未来预报、极端天气预警，自动提醒带伞、添减衣物，适配全球主流气象数据源，非常适合纳入「每日晨间简报」的自动化流程中。

**7. content-digest — 信息智能消化**
替代单一的newsletter/youtube技能，将长篇文章、播客、YouTube 长视频、播客音频自动转为核心要点与可执行行动清单，完美解决新手容易遇到的「信息过载」问题，大幅提升信息获取效率。

### 🥉 场景适配级
**8. GitHub — 开发辅助（面向开发者）**
查询仓库 Issue、审核 PR、解读代码、跟进 CI 流程、版本更新提醒，只要你从事代码开发或使用开源工具，就值得安装。

**9. GSC / GA4 — 网站增长分析（有个人/企业网站再安装）**
网站运营者必备技能，可查询关键词排名、页面收录状态、流量来源与用户行为数据，自动生成周度/月度分析报告，无网站用户可直接跳过。

**10. PDF Parser（markitdown）— 文档智能解析**
将 PDF、Word、PPT、图片等格式的文件转为可编辑文本，让 AI 助手秒读并总结核心内容，在收到几十页的项目材料、行业报告、合同时，能帮你节省大量的时间与精力。

## 五、技能组合拳：实现 1+1 > 2 的自动化效果
单个技能已经能解决具体问题，而多个技能的组合使用，才是 AI 助手远超传统工具的核心优势——它能将不同工具的数据串联起来，完成端到端的自动化流程与综合性分析。

### 组合 1：邮件 + 日历 + 待办
**指令示例**：帮我查看明天的所有会议，然后在我的邮箱中搜索相关的往来邮件，整理一份会前背景简报，再把需要我提前准备的事项生成高优先级待办，设置会议前1小时提醒。

助手会先查询日历，识别出明天的「合作方商务洽谈」会议，随后自动在邮箱中搜索该合作方的历史邮件往来，提炼核心沟通节点、待确认事项与关键背景信息，最终整理成一份完整的会前简报，同时生成待办清单并绑定提醒。

传统操作需要你：打开日历 → 查看会议信息 → 打开邮箱 → 搜索关键词 → 手动整理信息 → 创建待办 → 设置提醒。而现在，只需一句话即可完成。

### 组合 2：搜索 + 浏览器 + 内容摘要
**指令示例**：帮我搜索 "当下最佳无头 CMS 方案"，找到排名前三的行业分析文章，将文章中推荐的主流方案整理成对比表格，标注每个方案的核心优势、适用场景与定价模式。

助手会先执行全网搜索，找到高相关度的权威文章链接，再通过浏览器技能打开每一篇文章，提取核心参数、优劣势、适用场景等关键信息，最终整理成结构化的对比表格，甚至会给出针对性的选型建议。

### 组合 3：GSC + GA4 + 浏览器 + SEO 分析
**指令示例**：分析 kirkify.net 网站的 /generator 页面，告诉我它的搜索表现、用户行为数据，以及当前页面的内容状态、加载性能，最后给出一份分优先级的完整优化建议。

助手会分别调用三个技能完成全流程分析：
- 通过 GSC 技能查询页面的搜索表现（关键词排名、点击量、点击率 CTR）
- 通过 GA4 技能分析用户行为数据（页面停留时长、跳出率、转化路径）
- 通过浏览器技能打开页面，检查当前内容状态、加载速度、移动端适配情况

最终会为你输出一份完整的页面分析报告，并附带优先级排序的优化建议。

### 组合 4：社媒 + 搜索 + 内容创作
**指令示例**：帮我搜索当下TikTok上AI meme类内容的热门趋势，提取3个高流量选题，再结合kirkify.net的产品功能，写3条适配TikTok的短视频文案，设置好明天下午3点的发布提醒。

助手会先通过搜索技能抓取平台热门趋势，再通过浏览器技能提取产品核心卖点，完成文案创作后，通过待办+提醒技能设置发布计划，全程无需你手动切换多个平台、整理零散信息。

> 📌 最佳实践提示
> 技能组合的核心价值，是将你日常工作中重复、机械的多步操作，浓缩为一句指令。你可以从日常高频的工作流程入手，拆解其中需要用到的工具，再通过技能组合实现自动化，这是提升效率最直接的方式。

## 六、技能的日常管理
### 核心管理命令
```bash
# 查看已安装的所有技能（含版本、状态、使用频次）
openclaw skills list --detail

# 从 ClawdHub 安装/更新技能
clawdhub install <skill-name>    # 安装指定技能
clawdhub install <skill-name> --global # 全局安装
clawdhub update <skill-name>     # 更新单个技能
clawdhub update --all            # 批量更新所有已安装技能

# 在技能市场中搜索技能（支持按分类、评分筛选）
clawdhub search <关键词>

# 技能健康校验与状态管理
openclaw skills validate <skill-name> # 校验单个技能的配置完整性
openclaw skills validate --all # 全量校验所有已安装技能
openclaw skills disable <skill-name> # 临时禁用技能（无需卸载）
openclaw skills enable <skill-name> # 启用已禁用的技能
```

### 配置管理说明
每个技能的核心配置，通常写在该技能目录下的 SKILL.md 文件中（也可通过 `openclaw.json` 文件中的 `skills.entries.*` 配置项进行覆盖）。

技能目录默认路径为：`<workspace>/skills/<skill-name>/` 或 `~/.openclaw/skills/<skill-name>/`。

## ⚠️ 重要提示

最后必须提醒你一个核心原则：**技能并非安装越多越好。**

即便最新架构已优化了懒加载机制，每一个技能依然会增加助手的「认知负载」——每次对话时，它都需要匹配所有技能的适用场景，来判断该调用哪个技能解决你的问题。安装过多的技能，反而可能导致三个问题：
1.  响应速度变慢：需要处理的场景匹配逻辑大幅增加
2.  Token 消耗上升：复杂场景下会携带更多技能的说明信息，增加使用成本
3.  技能调用错误：技能过多时，可能出现选错技能、无法匹配正确工具的情况

**新手建议**：从你最核心需求对应的 3-5 个技能开始，用熟、用透之后，再根据新的需求逐步添加新技能。

这就像手机安装应用——安装 200 个应用但日常只用 20 个的人，手机运行速度一定会比只装 20 个高频应用的人慢。
