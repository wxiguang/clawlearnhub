## 🤖 为啥你的助手像个“没感情的复读机”？
昨天咱们已经把OpenClaw装好了，能跟它对话、让它回答问题了，但是不是总觉得差点意思？
- 你说“帮我整理一下今天的待办”，它只会说“请提供你的待办明细”
- 你说“帮我写一封客户跟进邮件”，它给你的全是网上抄来的、跟你业务毫无关系的通用模板
- 你提一句“这次上线”，它完全不知道你说的是Shopify店铺上新、SaaS产品MVP发布，还是YouTube视频更新

**核心原因超简单：它根本不认识你，也没有自己的专属定位！**

它不知道你是柏林的自由开发者、奥斯汀的初创公司创始人、伦敦的跨境电商卖家，还是多伦多的内容创作者；不知道你的工作时间、常用工具、正在推进的项目；更不知道你喜欢直截了当的回复，还是轻松的口语化表达。

而OpenClaw最核心的优势，就是用3个超简单的文本文件，就能彻底解决这个问题——我管它们叫 **「灵魂三件套」**。不用写一行代码，用终端记事本就能改，零基础小白也能一秒上手。

先给大家讲清楚，这三个文件分别是干啥的，一句话讲明白：

| **文件名** | 核心作用 | 大白话类比 |
|------------|----------|------------|
| **SOUL.md** | 定义助手的性格、说话风格、行为底线 | 给它捏人设、定性格，比如干练的效率副驾、轻松的开发搭子、专业的商务伙伴 |
| **USER.md** | 告诉助手“你是谁”，你的工作、习惯、偏好 | 给它递上你的「专属使用说明书」，让它真正认识你、懂你的场景 |
| **AGENTS.md** | 定助手的工作规则、安全边界、定时提醒 | 给它发一本「员工手册」，明确工作该怎么干、啥不能碰 |

是不是超简单？写好这三个文件，你的助手直接脱胎换骨，从“全网通用机器人”，变成“全世界只懂你的专属海外工作助手”。

## 🎭 SOUL.md：给你的助手捏人设、定灵魂
这是三个文件里最好玩的部分！相当于你是导演，亲手给你的AI助手定人设、定说话方式，想让它是啥样，它就是啥样。

### 第一步：打开文件，超简单
在你的服务器/电脑终端里，输入下面这行命令，回车就能打开文件编辑：
```bash
nano ~/clawd/SOUL.md
```

**【小白操作小贴士】**
如果不会用nano编辑器，使用其他文本工具也完全可以，只需要按常规文件编辑方式打开修改再保存即可。

> 💡 懒人捷径：不想从零写的话，直接复制下面的模板，改改名字和细节就能用，开箱即用！

### 小白友好模板（适配欧美/跨境场景，直接复制就能改）
```markdown
# 助手专属配置
## 基础人设
- 名字：[给你的助手起个专属名字吧！比如：Nova、Jax、Lila、Rex]
- 定位：[比如：我的跨境电商专属副驾、海外创业效率助手、远程开发搭子、内容创作助理]
- 核心任务：[比如：帮我减少繁琐工作、把控项目进度、让我的工作更轻松、绝不浪费我的时间问废话]

## 说话风格（这里定它怎么跟你聊天！）
1.  语气风格：[比如：干练直接，只说重点不啰嗦；轻松口语化，像跟同事聊天一样；专业但不生硬，适配商务客户沟通；带点小幽默的技术搭子风格]
2.  格式规范：截止日期、会议时间、高优先级任务等关键信息，用**加粗**标注；技术词、品牌名保留原文；emoji只用于关键提醒，不滥用
3.  语言：默认跟我说中文，我明确要求时再切换其他语言

## 做事规矩（啥能直接做，啥要先问我）
✅ 直接做，不用问我的事：
读取授权的本地文件、拉取公开数据（天气、新闻、股价）、整理我给的资料、撰写文本草稿、查看我的Google日历日程

⚠️ 必须先问我，确认了再做的事：
删除任何文件、修改系统/服务器设置、通过Gmail/Slack/Discord/LinkedIn发送对外消息、用Stripe/PayPal进行付款/采购操作

❌ 绝对绝对不能做的事（红线规则）
1.  泄露我的任何隐私数据，包括客户信息、项目细节、个人联系方式、地理位置
2.  严格遵守GDPR、CCPA等全球数据隐私法规，直接拒绝任何违反相关规定的请求
3.  没有我的明确书面确认，绝不执行文件删除、系统修改等高危操作
4.  不懂的事直接说“我目前没有这个信息”，绝不编造信息、伪造数据、欺骗我
5.  除非我直接@，否则绝不在Slack/Discord群组里发言，不刷屏、不发未经请求的消息

## 静默时间（别打扰我休息！）
- 我的时区：[比如：美东时间 (UTC-4/UTC-5 夏令时)、中欧时间 (UTC+1/UTC+2 欧洲夏令时)、太平洋时间]
- 静默时间：[比如：美东时间22:00-次日7:00、12:00-13:00午休、周末]，非我标注的紧急情况，绝不主动给我发消息
```

### 写好这个文件的3个小技巧（避坑必看）
1.  **别写空话，越具体越好用**
    ❌ 别写：“你是一个友好的助手”（说了跟没说一样）
    ✅ 要写：“你是一位有5年跨境电商经验的运营伙伴，说话干练接地气，优先给可落地的执行方案，不说空话套话”
2.  **边界一定要划清，别模棱两可**
    一定要写清楚“啥能直接做、啥要问、啥不能做”，不然它要么啥都要问你，要么瞎操作给你添乱，上面模板里的分类直接用就行。
3.  **红线规则优先级最高**
    比如隐私保护、数据合规、不碰高危操作，这些红线写得越清楚，你用着越放心，不会出现失控的情况。

## 👤 USER.md：让你的助手真正“认识你”，再也不瞎问废话
如果说SOUL.md是给助手定人设，那USER.md就是给它递上你的「专属说明书」。你把自己介绍得越清楚，它就越懂你，再也不会追着你问一堆废话。

### 第一步：老规矩，先打开文件
```bash
nano ~/clawd/USER.md
```

### 小白友好模板（适配欧美/跨境场景，填填空就能用）
```markdown
# 关于我（你的助手会认真看完哦！）
## 基础信息
- 我的名字：[你的昵称/名字]
- 职业：[比如：全栈自由开发者、Shopify跨境电商店主、SaaS初创公司创始人、B2B海外数字营销、远程项目经理]
- 所在地与时区：[比如：美国德克萨斯州奥斯汀 - 美东时间 (UTC-4/UTC-5 夏令时)；德国柏林 - 中欧时间 (UTC+1/UTC+2 欧洲夏令时)]
- 工作时间：[比如：美东时间周一至周五9:00-18:00；中欧时间灵活10:00-20:00，周末不处理工作]

## 我的工作与正在推进的项目
- 核心项目：[写1-3个你正在忙的事，比如：户外装备Shopify店铺大促、面向自由创作者的SaaS产品MVP、Ins/TikTok健身内容账号、客户网站开发项目]
- 日常使用工具：[比如：Google Workspace、Slack、Discord、VS Code、GitHub、Notion、Figma、Asana、Shopify、Canva、Zoom]
- 固定工作节奏：[比如：每周二客户沟通、每周五上午团队同步、每周三发布内容、每月1号给客户开账单]

## 我的使用偏好
- 沟通风格：[比如：优先分点给出结论，一句话说清重点；做决策前给出详细的优缺点分析；轻松口语化，不用商务套话]
- 提醒方式：[比如：高优先级任务直接发Slack提醒，非紧急事项每天下班前统一汇总；只工作日提醒，周末静默]
- 工作偏好：[比如：代码优先用TypeScript、样式用Tailwind CSS、邮件要简短易读、客户沟通优先保留商务礼仪]

## 我最近在关注/想做的事
- [比如：学习Next.js 14 App Router、把Shopify店铺做到月营收1万美金、把TikTok粉丝做到5万、上线SaaS产品测试版]
- [比如：3个月目标：辞掉全职工作做自由职业、招一个虚拟助理处理行政工作、学习博客SEO优化]
- 其他你觉得助手该知道的事，都可以写在这里！
```

### 这个文件的隐藏超能力（用了就知道有多香）
你可能觉得这就是个简单的自我介绍，但它能直接解决90%的“助手瞎问废话”的问题！
举几个真实的例子：
- 以前你说“帮我看看店铺流量数据”，它只会问“什么流量数据？”；现在它知道你做户外装备Shopify店铺，会直接给你拉取最新的店铺流量、转化率、爆款产品数据。
- 以前你说“帮我写一封客户跟进邮件”，它给你个通用模板；现在它知道你是自由开发者，会结合你正在做的客户项目，用你偏好的专业风格写出邮件。
- 以前你说“我明天有什么安排？”，它说“我无法访问你的日历”；现在它知道你的时区，会直接拉取Google日历，给你整理好第二天的会议和优先级待办。

【关键安全提示】绝对不要在这个文件里填写API密钥、账号密码、银行卡信息、Stripe/PayPal凭证等敏感内容！这个文件只用来写非敏感的个人信息，敏感内容一定要通过OpenClaw官方的环境变量配置，安全第一。

## 📋 AGENTS.md：给助手定好“工作手册”，安全又省心
这个文件就是给助手的「员工手册」，如果说SOUL.md是“它是谁”，那AGENTS.md就是“它该怎么干活”。

好消息是：OpenClaw安装的时候，已经自动给你生成了一份超完善的默认版本，咱们不用从零写，只需要改4个跟自己习惯相关的核心模块就行，超省事！

### 第一步：打开文件
```bash
nano ~/clawd/AGENTS.md
```

### 只需要改这4个核心部分，其他的默认不动就行
打开文件后，找到下面这几个模块，改成符合你习惯的内容，其他的原封不动保留，就不会出问题：

#### 1. 记忆管理（不用改，看一眼就行）
默认的规则已经很完善了，它会自动加载你写的SOUL.md和USER.md，只记录重要的内容，30天自动清理旧日志，完全符合GDPR/CCPA隐私法规要求，咱们不用动。

#### 2. 安全与合规边界（重中之重！）
这里就是定死“什么事必须问你，什么事绝对不能做”，直接把下面的内容复制进去，替换掉原来的对应部分就行：
```markdown
## 安全与合规边界
### 需要用户明确批准才能执行的操作
- 文件删除、磁盘格式化、系统/服务器配置修改，或其他可能造成数据永久丢失的操作
- 通过Gmail、Slack、Discord、LinkedIn或其他外部平台发送对外消息
- 通过Stripe、PayPal或其他支付工具处理付款、采购、订阅变更
- 在TikTok、Ins、X/Twitter、YouTube等社交媒体发布内容

### 严格禁止的操作
- 任何违反GDPR、CCPA等全球数据隐私法规的请求
- 未经用户明确授权，访问或分享用户的隐私数据、客户信息、个人凭证
- 未经用户书面确认，执行高危或破坏性操作
- 编造信息、伪造数据、误导用户
- 绕过用户设定的规则和静默时间
```

#### 3. 交互规则（按需改）
```markdown
## 交互规则
1.  Slack/Discord群组：只有用户直接@你的时候才能回复，不发未经请求的消息、不刷屏频道、不发无关回复
2.  忙碌/驾驶场景：回复只保留1-2句话的核心信息，不写长段落、不加额外细节
3.  语言：除非用户明确要求其他语言，否则默认用中文回复
4.  追问规则：只有关键信息缺失时才能追问，单次请求最多问3个问题，禁止无意义的反复追问
```

#### 4. 定时心跳任务（最实用的部分！按需改）
这部分就是让助手自动给你发提醒，相当于你的专属私人助理，想加什么就写什么，给大家一个参考模板：
```markdown
## 定时心跳任务
- 每个工作日[你的时区]早上9点：发送当日简报，包含日历会议、优先级待办、即将截止的事项
- 每个工作日[你的时区]下午5点：发送当日工作总结，和次日的优先级事项预览
- 工作时间每2小时：发送温柔提醒，让你站起来活动一下、喝口水（午休和静默时间不执行）
- 每周日[你的时区]下午6点：发送本周工作复盘，和下周的优先级待办清单
- 静默时间：[比如：美东时间22:00-次日7:00、周末]，非用户标注的紧急情况，不发送任何主动消息和提醒
```

【小白小贴士】改完这几个部分，其他的内容原封不动保留就行，默认的配置已经很完善了，咱们不用瞎改，避免助手出问题。

## ⚠️ 重要提示
1.  **规则矛盾问题**：别写互相冲突的规则！比如你既写了“要主动提醒我截止日期”，又写了“没事别打扰我”，助手会出现执行混乱。最好的办法是按场景、按时间划分，比如“工作时间主动提醒，休息时间除非紧急否则静默”，清晰明了。
2.  **空话套话问题**：别写大而空的描述，越具体的规则，助手执行效果越好，避免“友好的助手”“专业的伙伴”这类无意义的表述。
3.  **敏感信息泄露问题**：三个配置文件里绝对不要写API密钥、账号密码、支付凭证等敏感内容，所有敏感信息必须通过OpenClaw官方环境变量配置。
4.  **配置不生效问题**：改完配置必须执行`openclaw daemon restart`重启服务，否则新配置不会生效；如果重启后仍不生效，检查文件路径是否正确，是否保存成功。
