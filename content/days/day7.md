## ⚠️ 使用OpenClaw必须注意的关键问题
OpenClaw的能力越强，越需要我们守住安全和合规的底线。这些关键问题，直接决定了你的AI助手用得稳不稳、安不安全，每一条都要记牢！

### 1. 数据安全与隐私保护，是不可触碰的第一条红线🔒
OpenClaw会接入你的邮箱、日历、代码库、甚至电脑屏幕，这些核心隐私数据，必须做好万全防护：
- **API密钥绝对保密**：大模型API、平台接入密钥绝不泄露到公开代码、绝不分享给他人，定期更换密钥，开启IP白名单与权限限制；**绝对禁止将API密钥硬编码在配置文件、自定义技能代码中，禁止将包含密钥的文件提交到公开Git仓库、分享到公开社群**，避免密钥泄露导致的资产损失。
- **本地部署优先，数据自主掌控**：高敏感数据场景，优先选择本地/私人海外云服务器部署，所有数据全程留在自己手里。
- **最小权限原则**：给助手开通平台权限时，只开必要权限——只读邮件就不开修改权限，查看日历就不开日程删除权限，从根源规避风险；**平台接入的OAuth授权，优先选择只读权限，仅在必须执行写入/修改操作时，才开放对应细分权限，绝不默认勾选全量权限**。
- **合规适配当地法规**：严格遵守GDPR、CCPA等海外隐私法规，不超范围收集处理个人数据，敏感信息开启自动屏蔽。

### 2. 别让「过度自动化」给你挖坑🤦‍♂️
自动化能解放双手，但无底线的自动化，反而会带来大麻烦：
- **高风险操作必须人工确认**：正式商务邮件、代码库PR合并、账户付款等操作，必须让助手先出草稿、你确认后再执行，绝对禁止全自动操作。
- **AI是执行者，不是决策者**：它可以帮你整理数据、分析情况、给出建议，但最终决策一定要自己做。
- **定期清理无效任务**：每周花10分钟巡检心跳和Cron任务，删掉无用规则，避免「僵尸任务」无效消耗资源。

### 3. 控制好使用成本，别让API账单超支💰
OpenClaw本身免费开源，但大模型API调用会产生费用，做好这4点就能完美控本，精准预判开销、避免意外超支：
- **设置消费硬上限**：在大模型平台设置月度消费封顶、单次调用限额，超出自动停服，避免无限扣费。
  主流平台具体设置路径：
  - **OpenAI平台**：登录[OpenAI Platform](https://platform.openai.com) → 左侧菜单选择「Settings → Limits」→ 找到「Usage limits」板块 → 分别设置「Hard limit」（硬消费上限，达到后立即停用API）和「Soft limit」（软上限，达到后发送邮件提醒）→ 保存设置。
  - **Anthropic平台**：登录[Anthropic Console](https://console.anthropic.com) → 左侧菜单选择「Settings → Usage & Billing」→ 找到「Spend Limits」板块 → 开启「Monthly spend limit」→ 设置月度最大消费金额、提醒阈值 → 开启「Auto-pause at limit」（达到上限自动暂停API）→ 保存配置。
- **优化执行频率**：非工作时间调大心跳间隔，低频核对任务降低巡检频次，直接减少大半API成本。
- **开启缓存复用**：重复查询需求优先调用历史缓存结果，无需每次都重新调用大模型。
- **API消耗提前估算**：心跳机制、定时任务的API开销可精准预判，新手可参考以下基准数据（基于Claude 3.5 Sonnet/GPT-4o模型）：
  | 任务类型 | 执行频率 | 单次Token消耗 | 月度预估Token消耗 | 月度预估成本 |
  |----------|----------|---------------|--------------------|--------------|
  | 基础心跳巡检（仅邮件/日程检查） | 30分钟/次（工作日） | 1000-2000 Token | 35万-70万 Token | $0.35-$1.5 |
  | 全量心跳巡检（含网站/数据监控） | 15分钟/次（全天） | 2000-3000 Token | 576万-864万 Token | $5.8-$18 |
  | 晨间简报定时任务 | 1次/天 | 3000-5000 Token | 9万-15万 Token | $0.09-$0.3 |
  | 周报生成定时任务 | 1次/周 | 5000-10000 Token | 2万-4万 Token | $0.02-$0.08 |
  > 注：以上为基准估算，实际消耗取决于任务复杂度、上下文长度、模型选择；免费额度/低单价模型可进一步降低成本。

### 4. 做好备份，避免助手突然「失忆」💾
你的记忆文件、自动化规则、配置文件，都是花了大量时间打磨的核心资产：
- **定期备份核心文件**：每周备份一次`HEARTBEAT.md`、`MEMORY.md`、`SOUL.md`与Cron配置，服务器出问题也能一键恢复。
- **Git版本管理**：用Git管理配置文件，每次修改都提交版本，出问题可随时回滚到稳定版（提交前务必检查并排除包含API密钥的文件）。
- **多设备同步必做加密**：跨设备同步时，务必做好文件加密与权限管控，杜绝核心配置裸奔。

### 5. 永远保留「人工兜底」的能力✅
不管你的AI助手有多能干，你永远要比它更清楚事情的全貌：
- 重要邮件、代码、正式文件，一定要自己终审后再发出。
- 提前做好故障预案，API宕机、工具调用失败时，有手动处理的备用方案。
- 核心、高风险事项，一定要自己掌控最终环节。

## 🚀 3个核心方法，让你的AI助手越变越聪明
很多人用了几个月OpenClaw，助手还是「新手水平」；而有的人用了几周，助手就成了能独当一面的「金牌管家」。核心区别，就在于这3个持续优化的方法。

### 方法1：喂好「记忆系统」，让它真正懂你❤️
OpenClaw的核心优势，就是完全可控的记忆系统——你喂给它的信息越精准、越完整，它就越懂你，输出就越贴合预期。
- 每次和助手约定新规则、新偏好，都及时更新到`MEMORY.md`里，真正做到「教一次，记一辈子」。
- 在心跳任务里加入「每日记忆复盘」，让助手自动从当日笔记里提炼核心信息，更新到长期记忆，无需手动修改。
- 每半个月给记忆做「断舍离」，删掉过时信息，避免记忆库臃肿导致调用混乱。

### 方法2：建立「反馈闭环」，让它每次都比上一次更好📈
AI助手和人一样，需要正向反馈和纠错，才能持续进步，一个简单的闭环就能让它快速进化：
- 出错时不只是说「不对」，要明确告诉它「哪里错了、你想要什么、以后怎么改」，并让它把规则记入长期记忆。
- 做得好时一定要给正向反馈，明确告诉它「哪里做得好，以后按这个标准来」，强化正确的输出模式。
- 每周做一次「能力复盘」，和助手一起优化执行规则，形成「执行-反馈-优化-再执行」的正向循环。

### 方法3：解锁自定义技能，让它的能力没有边界🛠️
OpenClaw完全开源的特性，让它的能力上限远超封闭SaaS AI工具，你可以给它开发专属自定义技能，适配你独一无二的工作与生活场景：
- 先从全球社区1700+现成技能入手，跨境运营、股票盯盘、内容创作、旅行规划等场景，一键安装即可使用。
- 哪怕只有基础代码能力，也能快速开发专属技能，比如跨境店铺订单监控、播客内容剪辑、客户线索跟进，只要你能想到，它都能做到。
- 优化技能调用逻辑，告诉助手「什么场景用什么技能、调用规则是什么」，让它能自动适配场景调用对应能力。
